# Deep Researchでのデータ収集手順

## ステップ1: プロンプトの準備

1. `prompts/00_system-prompt.md` の内容をコピー
2. 各番号のプロンプト（01〜61）の内容をその下に貼り付ける
3. これをOpenAI Deep Researchに投入

## ステップ2: Deep Researchの実行

**推奨設定**: Deep Research (o1ベース) を使用

1つのプロンプトにつき1回のDeep Research実行。
各セッションで500〜1000件のJSONデータが出力される想定。

### 実行順序（推奨）

最初はデータ量が多い主要地域から:

```
01-08: 日本（8本）
09-14: 中国（6本）
15-17: 朝鮮半島（3本）
38-49: ヨーロッパ（12本）← ここが最大
27-31: 中東（5本）
50-56: 南北アメリカ（7本）
21-24: 南アジア（4本）
18-20: 東南アジア（3本）
25-26: 中央アジア（2本）
32-37: アフリカ（6本）
57-58: オセアニア（2本）
59-61: グローバル（3本）
```

## ステップ3: データの保存

Deep Researchの出力をコピーして、JSONファイルとして保存:

```
data-collection/raw/01_japan_ancient.json
data-collection/raw/02_japan_heian.json
...
```

**注意**: Deep Researchが ```json ``` ブロックで出力した場合、そのままコピペしてOK。
処理スクリプトが自動的にJSONブロックを抽出する。

## ステップ4: データ処理

```bash
cd ~/history-map
node scripts/process-data.js
```

処理結果:
- `data-collection/processed/all-events.json` - 全イベント統合データ
- `data-collection/processed/events-chunk-0.json` 〜 - 分割チャンク
- `data-collection/processed/manifest.json` - メタデータ
- `data-collection/processed/errors.log` - エラーログ

## ステップ5: サイトへの反映

処理済みデータをサイトに反映するには、サイトのアーキテクチャ更新後に
チャンクファイルを読み込む仕組みを使用する。

## Tips

- Deep Researchが途中で止まった場合は「続けてください」と送信
- 出力が不十分な場合は「もっと詳細に、特に○○について追加してください」と追加指示
- 1回のセッションで600件以上出ない場合は、テーマを絞って追加プロンプトを作成
- 座標が不正確な場合は、処理スクリプトのバリデーションで検出される

## 目標件数の達成に向けて

61本のプロンプトで合計約43,500件を想定。
不足分は以下の方法で補完:

1. 各プロンプトの出力が想定より少ない場合 → 追加の詳細プロンプトを作成
2. 特定地域・時代が薄い場合 → 補完プロンプトを追加
3. 品質チェック後に不足カテゴリがあれば → カテゴリ特化プロンプトを追加
